1. What is K-Means Clustering?

K-Means is an unsupervised machine learning algorithm used for clustering data into K distinct groups (clusters). It partitions a dataset into K clusters where each data point belongs to the cluster with the nearest centroid.

üéØ Key Characteristics:

  • Unsupervised Learning: No labeled data required
  • Partitioning Method: Divides data into non-overlapping groups
  • Distance-Based: Uses distance metrics (typically Euclidean)
  • Iterative: Refines clusters through multiple iterations
  • K is Predefined: Number of clusters must be specified beforehand

Mathematical Foundation

Objective Function (Minimize Within-Cluster Sum of Squares - WCSS):

J = Œ£(i=1 to K) Œ£(x‚ààCi) ||x - Œºi||¬≤

Where:
‚Ä¢ K = number of clusters
‚Ä¢ Ci = i-th cluster
‚Ä¢ x = data point in cluster Ci
‚Ä¢ Œºi = centroid of cluster Ci
‚Ä¢ ||x - Œºi||¬≤ = squared Euclidean distance
Euclidean Distance Formula (2D):

d(p, q) = ‚àö[(x‚ÇÇ - x‚ÇÅ)¬≤ + (y‚ÇÇ - y‚ÇÅ)¬≤]

Euclidean Distance Formula (n-dimensional):

d(p, q) = ‚àö[Œ£(i=1 to n)(qi - pi)¬≤]
Centroid Calculation:

Œºi = (1/|Ci|) Œ£(x‚ààCi) x

For 2D points: Œº = (mean of x-coordinates, mean of y-coordinates)

How K-Means Works - Algorithm Steps

1 Initialization: Randomly select K data points as initial centroids (or use other initialization methods like K-Means++)
2 Assignment Step: Assign each data point to the nearest centroid based on Euclidean distance
3 Update Step: Recalculate centroids as the mean of all points assigned to each cluster
4 Convergence Check: Repeat steps 2-3 until centroids no longer change (or changes are below a threshold) or maximum iterations reached

‚ö†Ô∏è Important Notes:

  • K-Means is sensitive to initial centroid selection
  • May converge to local optima instead of global optima
  • Assumes clusters are spherical and roughly equal-sized
  • Sensitive to outliers
  • Requires features to be on similar scales (normalization recommended)

6. Exam Tips & Common Mistakes

‚úÖ DO's:

  • Calculate distances to ALL centroids before making assignments
  • Show distance calculations clearly with formula substitution
  • Round intermediate calculations to 2 decimal places for clarity
  • Create a table for cluster assignments in each iteration
  • Verify convergence by comparing cluster assignments
  • Recalculate centroids using mean of all points in cluster
  • Label iterations clearly (Iteration 1, Iteration 2, etc.)

‚ùå DON'Ts / Common Mistakes:

  • Don't forget to calculate distance to BOTH/ALL centroids
  • Don't assign points before calculating all distances
  • Don't confuse x and y coordinates in distance formula
  • Don't forget to take square root in Euclidean distance
  • Don't stop after just one iteration (unless specified)
  • Don't use old centroids for next iteration calculations
  • Don't round too early - maintain precision in calculations

üéØ Quick Convergence Check:

Algorithm converges when ANY of these conditions is met:

  1. Cluster assignments don't change between iterations
  2. Centroids don't change (or change is below threshold like 0.01)
  3. Maximum iterations reached (if specified in problem)
